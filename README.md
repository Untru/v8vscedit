# 1С: Редактор конфигураций (v8vscedit)

`v8vscedit` — расширение для VS Code и Cursor, которое позволяет полноценно работать с конфигурациями и расширениями 1С:Предприятие прямо в редакторе: просматривать и менять метаданные, писать код, синхронизировать проект с базой, запускать 1С и работать с хранилищем конфигурации.

В качестве шаблона проекта используется структура `vanessa-bootstrap`: основная конфигурация хранится в `src/cf`, расширения — в `src/cfe/<ИмяРасширения>`, а параметры окружения задаются в `env.json`.

## Возможности

### Навигация по метаданным

- Дерево основной конфигурации и расширений в панели **1С: Редактор**.
- Поиск по дереву метаданных.
- Открытие XML-файлов объектов и модулей 1С из контекстного меню.
- Отображение собственных и заимствованных объектов расширений.
- Информация о поддержке конфигурации и контроль возможности редактирования.
- Подсветка объектов конфигурации по состоянию изменений в Git.

### Редактирование конфигурации и расширений

- Добавление объектов метаданных в конфигурацию и расширения.
- Удаление объектов метаданных с учётом ограничений поддержки и хранилища.
- Редактирование свойств объектов через отдельную панель свойств.
- Заимствование объектов основной конфигурации в расширение.
- Работа с модулями объектов, менеджеров, форм, команд, регистров, сервисов и общих модулей.

### Синхронизация с базой 1С

- Импорт основной конфигурации и расширений из информационной базы в XML-исходники.
- Обновление конфигурации или расширения в базе из текущих исходников.
- Автоматическое определение изменённых конфигураций по локальному хеш-кэшу.
- Обновление всех изменённых конфигураций одной кнопкой.
- Предложение обновить базу перед запуском тонкого клиента, если в исходниках есть незагруженные изменения.
- Удобный интерфейс настройки проекта: выбор платформы 1С, информационной базы, пользователя и параметров подключения.

### Запуск и окружение

- Запуск тонкого клиента 1С из проекта.
- Запуск конфигуратора 1С из проекта.
- Работа с автономным сервером 1С: настройка, запуск, остановка, перезапуск, открытие веб-клиента и просмотр лога.
- Автоматическая остановка и повторный запуск автономного сервера вокруг операций, которым нужен монопольный доступ к файловой базе.

### Хранилище конфигурации

- Подключение конфигурации или расширения к хранилищу.
- Создание и отключение хранилища.
- Захват, освобождение, помещение и получение объектов.
- Добавление и копирование пользователей хранилища.
- Выгрузка конфигурации из хранилища, отчёт по хранилищу и установка метки версии.
- Ограничение команд редактирования для объектов, которые нельзя менять из-за состояния хранилища.

### Код 1С и ИИ-агенты

- Поддержка языка 1С через интегрированный запуск `bsl-analyzer`.
- Автоматическая загрузка `bsl-analyzer`, если путь к нему не задан вручную.
- Проверка обновлений `bsl-analyzer` и команды перезапуска/просмотра лога.
- Установка 1С-скиллов для ИИ-агентов: Codex, Cursor, Claude Code, Cline, GitHub Copilot, Roo Code, Windsurf и других поддерживаемых клиентов.

## Требования

- VS Code или Cursor с API VS Code 1.85 и выше.
- Node.js 18 и выше, если расширение собирается из исходников.
- Установленная платформа 1С:Предприятие для команд запуска, импорта, обновления и хранилища.
- Информационная база 1С, доступная через строку подключения из `env.json`.
- Доступ к интернету для автоматической установки `bsl-analyzer` и скиллов ИИ-агентов, если они устанавливаются через команды расширения.

## Установка

### Из VSIX

1. Скачайте `v8vscedit-<версия>.vsix` из GitHub Releases.
2. В VS Code или Cursor выполните **Extensions → ... → Install from VSIX...**.

Можно установить и из командной строки:

```bash
code --install-extension v8vscedit-<версия>.vsix
cursor --install-extension v8vscedit-<версия>.vsix
```

### Из исходников

```bash
npm install
npm run compile
npm run build
```

Для разработки:

```bash
npm run watch
```

После этого откройте корень репозитория в VS Code или Cursor и нажмите `F5`, чтобы запустить Extension Development Host.

## Инициализация проекта 1С

Самый простой путь — открыть пустой каталог в VS Code или Cursor и перейти в панель **1С: Редактор**. Если проект ещё не инициализирован, расширение покажет стартовый интерфейс для создания структуры проекта.

Инициализация создаёт базовую структуру каталогов, `env.json`, настройки `bsl-analyzer` и правила `.gitignore`. Она специально разрешена только для пустого каталога, чтобы не перезаписать существующий проект.

После инициализации в той же панели доступны настройки проекта: выбор платформы 1С, информационной базы, пользователя и пароля. Эти параметры сохраняются в `env.json` в формате, совместимом с `vanessa-runner`.

Когда окружение настроено, можно импортировать конфигурацию и расширения из базы через кнопки синхронизации в панели **1С: Редактор**.

Основная конфигурация будет выгружена в `src/cf`, расширения — в `src/cfe/<ИмяРасширения>`.

## Требования к структуре проекта

Расширение считает каталог проектом 1С, если в корне рабочей области есть:

```text
project-root/
├── env.json
└── src/
    ├── cf/
    └── cfe/
```

`env.json` хранит параметры подключения:

```json
{
  "default": {
    "--ibconnection": "/F/path/to/base",
    "--db-user": "",
    "--db-pwd": "",
    "--path": "/path/to/1cv8",
    "--root": ".",
    "--workspace": "."
  }
}
```

После импорта или ручной выгрузки структура обычно выглядит так:

```text
project-root/
├── env.json
├── src/
│   ├── cf/
│   │   └── Configuration.xml
│   └── cfe/
│       └── МоеРасширение/
│           └── Configuration.xml
└── .v8vscedit/
    └── ...
```

Каталоги с конфигурациями определяются по файлу `Configuration.xml`. Основная конфигурация и расширения могут находиться глубже внутри рабочей области, но для команд синхронизации используется соглашение:

- `src/cf` — основная конфигурация;
- `src/cfe/<ИмяРасширения>` — расширение с именем, совпадающим с именем расширения в базе;
- `.v8vscedit` — служебные кэши расширения, их не нужно хранить в Git;
- `env.json` — локальные параметры подключения, его обычно не коммитят, потому что там могут быть пути, пользователи и пароли.

## Основные команды

- **Инициализировать проект 1С** — создаёт структуру нового проекта.
- **Настройки проекта** — выбирает платформу и информационную базу.
- **Импортировать конфигурации из базы** — выгружает XML-исходники из базы.
- **Обновить изменённые конфигурации** — загружает в базу только то, что изменилось в файлах.
- **Запустить тонкий клиент** — запускает 1С:Предприятие, при необходимости предлагает сначала обновить базу.
- **Запустить конфигуратор** — открывает 1С в режиме конфигуратора.
- **Автономный сервер: Настроить / Запустить / Остановить** — управляет автономным сервером проекта.
- **Установить ИИ-скилы 1С** — устанавливает набор скиллов для выбранных ИИ-агентов.

Большинство объектных команд доступны из контекстного меню узлов в панели **1С: Редактор**: открыть XML, открыть модуль, показать свойства, добавить, удалить, добавить в расширение, команды хранилища.

## Настройки BSL Analyzer

В настройках расширения доступны параметры:

- `v8vscedit.lsp.mode` — режим языковой поддержки 1С: `bsl-analyzer` или `off`.
- `v8vscedit.bslAnalyzer.autoUpdate` — проверять обновления `bsl-analyzer` при запуске.
- `v8vscedit.bslAnalyzer.path` — путь к установленному `bsl-analyzer`; если пусто, расширение скачает бинарник автоматически.
