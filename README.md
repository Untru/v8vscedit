# 1С: Редактор конфигураций (v8vscedit)

Расширение для VSCode / Cursor — редактор конфигураций и расширений 1С:Предприятие (XML-выгрузка). Навигация по дереву метаданных, языковой сервер BSL, работа с объектами по аналогии с конфигуратором 1С.

## Возможности

- Автоматически находит каталоги с `Configuration.xml` в рабочей папке
- Разделяет основную конфигурацию (CF) и расширения (CFE)
- Показывает объекты метаданных по группам: Справочники, Документы, Регистры сведений и т.д.
- Группа "Общие": Подсистемы, Общие модули, Роли, Общие формы и т.д.
- Ленивое раскрытие: реквизиты, ТЧ, формы, команды, макеты, измерения, ресурсы
- Для расширений помечает собственные `[свой]` и заимствованные `[заим.]` объекты
- Клик по объекту или дочернему элементу открывает соответствующий XML-файл
- Автообновление при изменении `Configuration.xml` (FileSystemWatcher)
- Кнопка «Обновить» на панели навигатора

## Требования

- Node.js >= 18
- npm
- VSCode >= 1.85 или Cursor
- `@vscode/vsce` для сборки VSIX: `npm install -g @vscode/vsce`

## Установка из GitHub Releases

1. Открой страницу **Releases** этого репозитория на GitHub.
2. Скачай файл расширения `v8vscedit-<версия>.vsix` из блока **Assets**.
3. Установи расширение из скачанного файла:

```bash
# VSCode
code --install-extension "C:\path\to\v8vscedit-<версия>.vsix"

# Cursor
cursor --install-extension "C:\path\to\v8vscedit-<версия>.vsix"
```

Альтернативно можно установить через UI редактора: **Extensions → ... → Install from VSIX...**

## Установка зависимостей

```bash
npm install
```

## Сборка

```bash
# Продакшн-сборка (минифицированный bundle в dist/)
npm run build

# Watch-режим для разработки (пересборка при изменениях)
npm run watch
```

## Отладка в VSCode / Cursor

1. Открыть корень проекта `c:\Projects\1CVCONF` в редакторе
2. Нажать `F5` — откроется **Extension Development Host** с уже загруженным расширением
3. Панель **1С Редактор** появится в Activity Bar слева (иконка документа)
4. После изменений в коде нажать `Ctrl+Shift+F5` для перезапуска

> Для watch-режима задача сборки запускается автоматически при F5 (`tasks.json`).

## Тестирование

```bash
# Компиляция + запуск unit-тестов
npm test
```

Тесты покрывают:

- `ConfigFinder` — обнаружение CF и CFE в каталоге `example/`
- `ConfigParser` — корректность парсинга `Configuration.xml` и объектных XML (`ев_Заказ`, `ев_ВидыРолейУчастников`, `ев_УчастникиЗаказа`)
- Проверка дочерних элементов: реквизиты, формы, значения перечислений, измерения, ресурсы

## Сборка VSIX и установка

```bash
# Собрать пакет (создаст файл v8vscedit-x.x.x.vsix)
vsce package

# Установить в VSCode
code --install-extension v8vscedit-*.vsix

# Установить в Cursor
cursor --install-extension v8vscedit-*.vsix
```

## Как работает дерево

| Тип корневого узла | Метка |
|---|---|
| Основная конфигурация | `Конфигурация: УправлениеНебольшойФирмой v3.0.13.210` |
| Расширение | `EVOLC` (просто имя расширения) |

Структура внутри узла повторяет конфигуратор 1С:
- **Общие** → Подсистемы, Общие модули, Роли, Определяемые типы, ...
- **Справочники** → каждый справочник → Реквизиты, Табличные части, Формы, ...
- **Документы**, **Перечисления**, **Регистры сведений**, ...

Для расширений собственные объекты (с префиксом `ев_`) помечаются `[свой]`, заимствованные (без префикса) — `[заим.]`.
