# Тестирование визуального редактора форм

## Подготовка

### 1. Сборка и установка

```bash
cd C:/Users/Pavel/v8vscedit-agent-25

# Сборка
npm run build

# Упаковка в VSIX
npx @vscode/vsce package --allow-missing-repository

# Установка в VS Code
code --install-extension v8vscedit-*.vsix --force
```

### 2. Тестовые данные

Нужен workspace с XML-выгрузкой конфигурации 1С, содержащей формы. Подходящие репозитории:

| Репозиторий | Путь |
|-------------|------|
| pz_unf | `D:\EDT\repos\pz_unf\src\cf` |
| Любая задача EDT | `D:\EDT\tasks\<task>\src\cf` |

Откройте папку с выгрузкой в VS Code:
```bash
code D:\EDT\repos\pz_unf\src\cf
```

### 3. Примеры Form.xml для тестирования

| Форма | Путь | Особенности |
|-------|------|-------------|
| Форма списка | `AccumulationRegisters/*/Forms/ФормаСписка/Ext/Form.xml` | Table, колонки |
| Форма элемента | `Catalogs/*/Forms/ФормаЭлемента/Ext/Form.xml` | Группы, поля |
| Форма документа | `Documents/*/Forms/ФормаДокумента/Ext/Form.xml` | Сложная: Pages, таблицы, кнопки |
| Общая форма | `CommonForms/*/Ext/Form.xml` | Простая структура |

## Тест-кейсы

### TC-01: Открытие формы в визуальном редакторе

**Шаги:**
1. Найти любой `Form.xml` в файловом обозревателе VS Code
2. Правый клик → **Open With...** → **Визуальный редактор формы 1С**

**Ожидание:**
- Открывается вкладка с тремя панелями
- Левая: дерево элементов с иконками
- Центр: визуальное превью формы
- Правая: "Выберите элемент формы"

### TC-02: Дерево элементов

**Шаги:**
1. Открыть Form.xml в визуальном редакторе
2. Кликнуть на стрелку ▸ рядом с группой

**Ожидание:**
- Группы раскрываются/сворачиваются
- Показываются вложенные элементы
- По умолчанию раскрыто 2 уровня
- У каждого элемента видно имя и иконку типа

### TC-03: Выбор элемента

**Шаги:**
1. Кликнуть на элемент в дереве
2. Кликнуть на элемент в превью

**Ожидание:**
- Элемент подсвечивается и в дереве, и в превью одновременно
- В правой панели появляются свойства: Имя, Тип, ID, Путь к данным, Заголовок и т.д.

### TC-04: Превью различных типов элементов

**Проверить наличие и корректный рендер:**

| Тип | Как выглядит |
|-----|-------------|
| `UsualGroup` (Vertical) | Вертикальный контейнер |
| `UsualGroup` (Horizontal) | Горизонтальный контейнер |
| `InputField` | Подпись + пустое поле ввода |
| `Button` | Синяя кнопка с текстом |
| `Table` | Заголовки колонок + пустое тело |
| `Pages` | Вкладки сверху + содержимое первой |
| `CheckBoxField` | Чекбокс + подпись |
| `LabelDecoration` | Текстовая надпись |

### TC-05: Drag-and-drop в дереве

**Шаги:**
1. Захватить элемент в дереве мышкой (drag)
2. Перетащить над другим элементом
3. Отпустить (drop)

**Ожидание:**
- При перетаскивании появляется индикатор позиции (синяя полоска сверху или снизу)
- После drop: элемент перемещается на новую позицию
- Превью обновляется
- В табе появляется точка (dirty state)

### TC-06: Редактирование свойств

**Шаги:**
1. Выбрать элемент в дереве
2. В панели свойств изменить:
   - Группировку (Vertical → Horizontal) для UsualGroup
   - Видимость (убрать чекбокс)
   - Ширину (ввести число)

**Ожидание:**
- Превью обновляется сразу после изменения
- Группа меняет направление компоновки
- Невидимый элемент показывается полупрозрачным с пунктирной рамкой
- В табе появляется точка (dirty state)

### TC-07: Удаление элемента

**Шаги:**
1. Выбрать элемент
2. Нажать кнопку "Удалить элемент" в нижней части панели свойств
   (или нажать клавишу Delete)

**Ожидание:**
- Элемент исчезает из дерева и превью
- Панель свойств очищается
- В табе появляется точка

### TC-08: Сохранение (Ctrl+S)

**Шаги:**
1. Внести любые изменения (drag, edit, delete)
2. Нажать Ctrl+S

**Ожидание:**
- Точка в табе исчезает
- Form.xml на диске обновлён
- При повторном открытии файла — изменения сохранены

### TC-09: Revert

**Шаги:**
1. Внести изменения
2. File → Revert File (или Ctrl+Shift+P → "File: Revert File")

**Ожидание:**
- Все изменения отменены
- Форма показывает исходное состояние

### TC-10: Внешние изменения

**Шаги:**
1. Открыть Form.xml в визуальном редакторе
2. Не вносить изменений
3. Отредактировать тот же Form.xml в текстовом редакторе и сохранить

**Ожидание:**
- Визуальный редактор автоматически обновляется

## Известные ограничения (v1)

- **Title** в свойствах — обновляет простое текстовое значение, не локализованную строку `<v8:item>`
- Нет undo/redo (Ctrl+Z не отменяет отдельные действия, только Revert целиком)
- Нет добавления новых элементов (только перемещение и удаление существующих)
- Round-trip XML может немного отличаться форматированием от оригинала
- Нет контекстного меню в превью
